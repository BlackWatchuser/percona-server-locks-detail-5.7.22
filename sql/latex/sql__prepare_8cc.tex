\subsection{Detailed Description}
This file contains the implementation of prepared statements.

When one prepares a statement\+:


\begin{DoxyItemize}
\item Server gets the query from client with command \textquotesingle{}C\+O\+M\+\_\+\+S\+T\+M\+T\+\_\+\+P\+R\+E\+P\+A\+RE\textquotesingle{}; in the following format\+: \mbox{[}C\+O\+M\+\_\+\+S\+T\+M\+T\+\_\+\+P\+R\+E\+P\+A\+RE\+:1\mbox{]} \mbox{[}query\mbox{]}
\item Parse the query and recognize any parameter markers \textquotesingle{}?\textquotesingle{} and store its information list in lex-\/$>$param\+\_\+list
\item Allocate a new statement for this prepare; and keep this in \textquotesingle{}thd-\/$>$stmt\+\_\+map\textquotesingle{}.
\item Without executing the query, return back to client the total number of parameters along with result-\/set metadata information (if any) in the following format\+: \begin{DoxyVerb}    [STMT_ID:4]
    [Column_count:2]
    [Param_count:2]
    [Params meta info (stubs only for now)]  (if Param_count > 0)
    [Columns meta info] (if Column_count > 0)\end{DoxyVerb}

\end{DoxyItemize}

During prepare the tables used in a statement are opened, but no locks are acquired. Table opening will block any D\+DL during the operation, and we do not need any locks as we neither read nor modify any data during prepare. Tables are closed after prepare finishes.

When one executes a statement\+:


\begin{DoxyItemize}
\item Server gets the command \textquotesingle{}C\+O\+M\+\_\+\+S\+T\+M\+T\+\_\+\+E\+X\+E\+C\+U\+TE\textquotesingle{} to execute the previously prepared query. If there are any parameter markers, then the client will send the data in the following format\+: \begin{DoxyVerb}    [COM_STMT_EXECUTE:1]
    [STMT_ID:4]
    [NULL_BITS:(param_count+7)/8)]
    [TYPES_SUPPLIED_BY_CLIENT(0/1):1]
    [[length]data]
    [[length]data] .. [[length]data].\end{DoxyVerb}
 (Note\+: Except for string/binary types; all other types will not be supplied with length field)
\item If it is a first execute or types of parameters were altered by client, then setup the conversion routines.
\item Assign parameter items from the supplied data.
\item Execute the query without re-\/parsing and send back the results to client
\end{DoxyItemize}

During execution of prepared statement tables are opened and locked the same way they would for normal (non-\/prepared) statement execution. Tables are unlocked and closed after the execution.

When one supplies long data for a placeholder\+:


\begin{DoxyItemize}
\item Server gets the long data in pieces with command type \textquotesingle{}C\+O\+M\+\_\+\+S\+T\+M\+T\+\_\+\+S\+E\+N\+D\+\_\+\+L\+O\+N\+G\+\_\+\+D\+A\+TA\textquotesingle{}.
\item The packet received will have the format as\+: \mbox{[}C\+O\+M\+\_\+\+S\+T\+M\+T\+\_\+\+S\+E\+N\+D\+\_\+\+L\+O\+N\+G\+\_\+\+D\+A\+TA\+:1\mbox{]}\mbox{[}S\+T\+M\+T\+\_\+\+ID\+:4\mbox{]}\mbox{[}parameter\+\_\+number\+:2\mbox{]}\mbox{[}data\mbox{]}
\item data from the packet is appended to the long data value buffer for this placeholder.
\item It\textquotesingle{}s up to the client to stop supplying data chunks at any point. The server doesn\textquotesingle{}t care; also, the server doesn\textquotesingle{}t notify the client whether it got the data or not; if there is any error, then it will be returned at statement execute. 
\end{DoxyItemize}
