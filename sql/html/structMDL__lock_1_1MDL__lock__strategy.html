Detailed Description

Helper struct which defines how different types of locks are handled for a specific MDL_lock. In practice we use only two strategies: "scoped" lock strategy for locks in GLOBAL, COMMIT, TABLESPACE and SCHEMA namespaces and "object" lock strategy for all other namespaces.

◆ m_waiting_incompatible

bitmap_t MDL_lock::MDL_lock_strategy::m_waiting_incompatible[4][MDL_TYPE_END]

Arrays of bitmaps which elements specify which waiting locks are incompatible with the type of lock being requested. Basically, each array defines priorities between lock types. We need 4 separate arrays since in order to prevent starvation for some of lock request types, we use different priority matrices: 0) in "normal" situation. 1) in situation when the number of successively granted "piglet" requests exceeds the max_write_lock_count limit. 2) in situation when the number of successively granted "hog" requests exceeds the max_write_lock_count limit. 3) in situation when both "piglet" and "hog" counters exceed limit.
