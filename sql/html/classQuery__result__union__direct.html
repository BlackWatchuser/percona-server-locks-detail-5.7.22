Detailed Description

UNION result that is passed directly to the receiving Query_result without filling a temporary table.

Function calls are forwarded to the wrapped Query_result, but some functions are expected to be called only once for each query, so they are only executed for the first query block in the union (except for send_eof(), which is executed only for the last query block).

This Query_result is used when a UNION is not DISTINCT and doesn't have a global ORDER BY clause.

See also
st_select_lex_unit::prepare().
