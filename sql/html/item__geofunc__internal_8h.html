◆ handle_gis_exception()

void handle_gis_exception ( const char *  funcname)

Handle a GIS exception of any type.

This function constitutes the exception handling barrier between Boost.Geometry and MySQL code. It handles all exceptions thrown in GIS code and raises the corresponding error in MySQL.

Pattern for use in other functions:

try
{
something_that_throws();
}
catch (...)
{
handle_gis_exception("st_foo");
}

Other exception handling code put into the catch block, before or after the call to handle_gis_exception(), must not throw exceptions.

Parameters
funcnameFunction name for use in error message

Simplify multi-geometry data. If str contains a multi-geometry or geometry collection with one component, the component is made as content of str. If str contains a nested geometry collection, the effective concrete geometry object is returned.

Parameters
strA string buffer containing a GEOMETRY byte string.
[out]result_bufferif not NULL and if a simplification is to be done, we will copy the GEOMETRY byte string from str into result_buffer and simplify the one stored in result_buffer, and the one in str is intact. If there is any data in result_buffer before calling this function, it is overwritten; If no simplification done, the result_buffer is intact if it is provided.
Returns
