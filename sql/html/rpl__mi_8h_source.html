33 /*****************************************************************************
34  Replication IO Thread
35 
36  Master_info contains:
37  - information about how to connect to a master
38  - current master log name
39  - current master log offset
40  - misc control variables
41 
42  Master_info is initialized once from the master.info repository if such
43  exists. Otherwise, data members corresponding to master.info fields
44  are initialized with defaults specified by master-* options. The
45  initialization is done through mi_init_info() call.
46 
47  Logically, the format of master.info repository is presented as follows:
48 
49  log_name
50  log_pos
51  master_host
52  master_user
53  master_pass
54  master_port
55  master_connect_retry
56 
57  To write out the contents of master.info to disk a call to flush_info()
58  is required. Currently, it is needed every time we read and queue data
59  from the master.
60 
61  To clean up, call end_info()
62 
63 *****************************************************************************/
