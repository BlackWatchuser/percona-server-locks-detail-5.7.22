302  /*
303  Push condition down to the table handler.
304  SYNOPSIS
305  cond_push()
306  cond Condition to be pushed. The condition tree must not be
307  modified by the by the caller.
308  RETURN
309  The 'remainder' condition that caller must use to filter out records.
310  NULL means the handler will not return rows that do not match the
311  passed condition.
312  NOTES
313  The pushed conditions form a stack (from which one can remove the
314  last pushed condition using cond_pop).
315  The table handler filters out rows using (pushed_cond1 AND pushed_cond2
316  AND ... AND pushed_condN)
317  or less restrictive condition, depending on handler's capabilities.
318 
319  handler->reset() call empties the condition stack.
320  Calls to rnd_init/rnd_end, index_init/index_end etc do not affect the
321  condition stack.
322  The current implementation supports arbitrary AND/OR nested conditions
323  with comparisons between columns and constants (including constant
324  expressions and function calls) and the following comparison operators:
325  =, !=, >, >=, <, <=, like, "not like", "is null", and "is not null".
326  Negated conditions are supported by NOT which generate NAND/NOR groups.
327  */
328  const Item *cond_push(const Item *cond);
329  /*
330  Pop the top condition from the condition stack of the handler instance.
331  SYNOPSIS
332  cond_pop()
333  Pops the top if condition stack, if stack is not empty
334  */
335  void cond_pop();
