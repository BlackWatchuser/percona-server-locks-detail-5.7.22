1983  SYNOPSIS

1986  RETURN VALUE
1987  TRUE Partition function not accepted

1989

1991  check_partition_func_processor is used to check if a partition function
1992  uses an allowed function. An allowed function will always ensure that
1993  X=Y guarantees that also part_function(X)=part_function(Y) where X is
1994  a set of partition fields and so is Y. The problems comes mainly from
1995  character sets where two equal strings can be quite unequal. E.g. the
1996  german character for double s is equal to 2 s.
1997 
1998  The default is that an item is not allowed
1999  in a partition function. Allowed functions
2000  can never depend on server version, they cannot depend on anything
2001  related to the environment. They can also only depend on a set of
2002  fields in the table itself. They cannot depend on other tables and
2003  cannot contain any queries and cannot contain udf's or similar.
2004  If a new Item class is defined and it inherits from a class that is
2005  allowed in a partition function then it is very important to consider
2006  whether this should be inherited to the new class. If not the function
2007  below should be defined in the new Item class.
2008

2010  If the partition function contains any multi-byte collations then
2011  the function check_part_func_fields will report an error on the
2012  partition function independent of what functions are used. So the
2013  only character sets allowed are single character collation and
2014  even for those only a limited set of functions are allowed. The
2015  problem with multi-byte collations is that almost every string
2016  function has the ability to change things such that two strings
2017  that are equal will not be equal after manipulated by a string
2018  function. E.g. two strings one contains a double s, there is a
2019  special german character that is equal to two s. Now assume a
2020  string function removes one character at this place, then in
2021  one the double s will be removed and in the other there will
2022  still be one s remaining and the strings are no longer equal
2023  and thus the partition function will not sort equal strings into
2024  the same partitions.

2027  check that the field types are valid, next check that the partition
2028  function is valid. The current set of partition functions valid

2030  fields.
