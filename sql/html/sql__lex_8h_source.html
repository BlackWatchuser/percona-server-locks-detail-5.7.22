385 /*
386  Class st_select_lex_unit represents a query expression.
387  Class st_select_lex represents a query block.
388  A query expression contains one or more query blocks (more than one means
389  that we have a UNION query).
390  These classes are connected as follows:
391  Both classes have a master, a slave, a next and a prev field.
392  For class st_select_lex, master and slave connect to objects of type
393  st_select_lex_unit, whereas for class st_select_lex_unit, they connect
394  to st_select_lex.
395  master is pointer to outer node.
396  slave is pointer to the first inner node
397 
398  neighbors are two st_select_lex or st_select_lex_unit objects on
399  the same level.
400 
401  The structures are linked with the following pointers:
402  - list of neighbors (next/prev) (prev of first element point to slave
403  pointer of outer structure)
404  - For st_select_lex, this is a list of query blocks.
405  - For st_select_lex_unit, this is a list of subqueries.
406 
407  - pointer to outer node (master), which is
408  If this is st_select_lex_unit
409  - pointer to outer select_lex.
410  If this is st_select_lex
411  - pointer to outer st_select_lex_unit.
412 
413  - pointer to inner objects (slave), which is either:
414  If this is an st_select_lex_unit:
415  - first query block that belong to this query expression.
416  If this is an st_select_lex
417  - first query expression that belong to this query block (subqueries).
418 
419  - list of all st_select_lex objects (link_next/link_prev)
420  This is to be used for things like derived tables creation, where we
421  go through this list and create the derived tables.
422 
423  If query expression contain several query blocks (UNION now,
424  INTERSECT etc later) then it has a special select_lex called
425  fake_select_lex. It used for storing global parameters (like ORDER BY,
426  LIMIT) and executing union.
427  Subqueries used in global ORDER BY clause will be attached to this
428  fake_select_lex, which will allow them to correctly resolve fields of
429  the containing UNION and outer selects.
430 
431  For example for following query:
432 
433  select *
434  from table1
435  where table1.field IN (select * from table1_1_1 union
436  select * from table1_1_2)
437  union
438  select *
439  from table2
440  where table2.field=(select (select f1 from table2_1_1_1_1
441  where table2_1_1_1_1.f2=table2_1_1.f3)
442  from table2_1_1
443  where table2_1_1.f1=table2.f2)
444  union
445  select * from table3;
446

448 
449  select1: (select * from table1 ...)
450  select2: (select * from table2 ...)
451  select3: (select * from table3)
452  select1.1.1: (select * from table1_1_1)
453  ...
454 
455  main unit
456  fake0
457  select1 select2 select3
458  |^^ |^
459  s||| ||master
460  l||| |+---------------------------------+
461  a||| +---------------------------------+|
462  v|||master slave ||
463  e||+-------------------------+ ||
464  V| neighbor | V|
465  unit1.1<+==================>unit1.2 unit2.1
466  fake1.1
467  select1.1.1 select 1.1.2 select1.2.1 select2.1.1
468  |^
469  ||
470  V|
471  unit2.1.1.1
472  select2.1.1.1.1
473 
474 
475  relation in main unit will be following:
476  (bigger picture for:
477  main unit
478  fake0
479  select1 select2 select3
480  in the above picture)
481 
482  main unit
483  |^^^^|fake_select_lex
484  |||||+--------------------------------------------+
485  ||||+--------------------------------------------+|
486  |||+------------------------------+ ||
487  ||+--------------+ | ||
488  slave||master | | ||
489  V| neighbor | neighbor | master|V
490  select1<========>select2<========>select3 fake0
491 
492  list of all select_lex will be following (as it will be constructed by
493  parser):
494 
495  select1->select2->select3->select2.1.1->select 2.1.2->select2.1.1.1.1-+

497  +---------------------------------------------------------------------+
