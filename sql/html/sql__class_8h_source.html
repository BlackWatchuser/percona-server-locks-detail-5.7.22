4874  /*
4875  Send the status of the current statement execution over network.
4876 
4877  In MySQL, there are two types of SQL statements: those that return
4878  a result set and those that return status information only.
4879 
4880  If a statement returns a result set, it consists of 3 parts:
4881  - result set meta-data
4882  - variable number of result set rows (can be 0)
4883  - followed and terminated by EOF or ERROR packet
4884 
4885  Once the client has seen the meta-data information, it always
4886  expects an EOF or ERROR to terminate the result set. If ERROR is
4887  received, the result set rows are normally discarded (this is up
4888  to the client implementation, libmysql at least does discard them).
4889  EOF, on the contrary, means "successfully evaluated the entire
4890  result set". Since we don't know how many rows belong to a result
4891  set until it's evaluated, EOF/ERROR is the indicator of the end
4892  of the row stream. Note, that we can not buffer result set rows
4893  on the server -- there may be an arbitrary number of rows. But
4894  we do buffer the last packet (EOF/ERROR) in the Diagnostics_area and
4895  delay sending it till the very end of execution (here), to be able to
4896  change EOF to an ERROR if commit failed or some other error occurred
4897  during the last cleanup steps taken after execution.
4898 
4899  A statement that does not return a result set doesn't send result
4900  set meta-data either. Instead it returns one of:
4901  - OK packet
4902  - ERROR packet.
4903  Similarly to the EOF/ERROR of the previous statement type, OK/ERROR
4904  packet is "buffered" in the Diagnostics Area and sent to the client
4905  in the end of statement.
4906

4908  sending of data to virtual Protocol::send_{ok,eof,error}. This
4909  allows for implementation of protocols that "intercept" ok/eof/error
4910  messages, and store them in memory, etc, instead of sending to
4911  the client.
4912 
4913  @pre The Diagnostics Area is assigned or disabled. It can not be empty
4914  -- we assume that every SQL statement or COM_* command
4915  generates OK, ERROR, or EOF status.
4916 
4917  @post The status information is encoded to protocol format and sent to the
4918  client.
4919 
4920  @return We conventionally return void, since the only type of error
4921  that can happen here is a NET (transport) error, and that one
4922  will become visible when we attempt to read from the NET the
4923  next command.
4924  Diagnostics_area::is_sent is set for debugging purposes only.
4925  */
