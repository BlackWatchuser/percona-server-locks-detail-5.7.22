◆ read_packet()

int Protocol_callback::read_packet ( )
virtual

Forces read of packet from the connection

Returns
bytes read -1 failure

Practically does nothing. Returns -1, error, for the case this will be called. It should happen. read_packet() is called by get_command() which in turn is called by do_command() in sql_parse. After that COM_DATA is filled with proper info that in turn is passed to dispatch_command(). The Command service doesn't use do_command() but dispatch_command() and passes COM_DATA directly from the user(plugin).

Returns
-1 failure
