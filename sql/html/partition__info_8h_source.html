235  /*
236  Bitmaps of partitions used by the current query.
237  * read_partitions - partitions to be used for reading.
238  * lock_partitions - partitions that must be locked (read or write).
239  Usually read_partitions is the same set as lock_partitions, but
240  in case of UPDATE the WHERE clause can limit the read_partitions set,
241  but not neccesarily the lock_partitions set.
242  Usage pattern:
243  * Initialized in ha_partition::open().
244  * read+lock_partitions is set according to explicit PARTITION,
245  WL#5217, in open_and_lock_tables().
246  * Bits in read_partitions can be cleared in prune_partitions()
247  in the optimizing step.
248  (WL#4443 is about allowing prune_partitions() to affect lock_partitions
249  and be done before locking too).
250  * When the partition enabled handler get an external_lock call it locks
251  all partitions in lock_partitions (and remembers which partitions it
252  locked, so that it can unlock them later). In case of LOCK TABLES it will
253  lock all partitions, and keep them locked while lock_partitions can
254  change for each statement under LOCK TABLES.
255  * Freed at the same time item_free_list is freed.
256  */
